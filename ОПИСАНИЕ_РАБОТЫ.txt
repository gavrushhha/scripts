================================================================================
ОПИСАНИЕ РАБОТЫ СКРИПТА FileMonitor.vbs
================================================================================

ОБЩЕЕ ОПИСАНИЕ
--------------
FileMonitor.vbs - это скрипт для автоматического мониторинга папки и копирования
новых файлов на сетевой диск с проверкой целостности и поддержкой различных
сценариев работы (локальный диск, сетевой диск, UNC путь с авторизацией).


АРХИТЕКТУРА СКРИПТА
-------------------

1. ГЛОБАЛЬНЫЕ ПЕРЕМЕННЫЕ
   - fileQueue (Dictionary) - очередь файлов для обработки
   - fileSignatures (Dictionary) - сигнатуры обработанных файлов (размер + дата)
   - config (Dictionary) - конфигурация из Config.ini
   - isNetworkDriveMapped (Boolean) - флаг подключенного сетевого диска
   - queueIndex (Integer) - счетчик для нумерации файлов в очереди

6. СТАБИЛИЗАЦИЯ ФАЙЛА (WaitForFileStable)
   ┌─────────────────────────────────────┐
   │ Цель: убедиться, что файл           │
   │       перестал изменяться           │
   │       (без вычисления хэша)         │
   └────────────┬────────────────────────┘
                │
   ┌────────────▼────────────────────────┐
   │ ЦИКЛ:                                │
   │ 1. Чтение размера файла             │
   │ 2. Чтение даты изменения            │
   │ 3. Сравнение с предыдущими          │
   │ 4. Если не изменились →             │
   │    ждём FileStabilizationMs         │
   │ 5. Если изменились → сброс таймера  │
   └─────────────────────────────────────┘

   Это решает проблему:
   - Файл сохраняется постепенно
   - Файл изменяется и сохраняется снова
   - Файл удаляется и создаётся заново


7. ЗАЩИТА ОТ ПЕРЕЗАПИСИ (EnsureNoOverwriteDestPath)
   ┌─────────────────────────────────────┐
   │ 1. Проверка: файл уже существует?    │
   │    Если нет → вернуть исходный путь  │
   └────────────┬────────────────────────┘
                │
   ┌────────────▼────────────────────────┐
   │ 2. Если существует:                  │
   │    - Извлечь имя и расширение       │
   │    - Добавить суффикс:              │
   │      __YYYYMMDD_HHMMSS              │
   │    - Вернуть новый путь             │
   └─────────────────────────────────────┘

   Пример:
   Исходный: file.txt
   Результат: file__20240115_143025.txt

   Это гарантирует, что:
   - Старый файл не перезапишется
   - Новый файл сохранится с уникальным именем

ОСОБЕННОСТИ РАБОТЫ
------------------

1. БЕЗ ХЭШИРОВАНИЯ
   - Не используется MD5/SHA для проверки файлов
   - Используется сигнатура: размер + дата изменения
   - Это снижает нагрузку на систему

2. СТАБИЛИЗАЦИЯ ФАЙЛОВ
   - Ожидание, пока файл перестанет изменяться
   - Проверка размера и даты изменения
   - Защита от копирования незавершённых файлов

3. ЗАЩИТА ОТ ПЕРЕЗАПИСИ
   - Если файл с таким именем уже существует в назначении
   - Создаётся версия с временной меткой
   - Старый файл никогда не перезаписывается

4. РАБОТА С СЕТЕВЫМИ ДИСКАМИ
   - Поддержка UNC путей (\\server\share)
   - Авторизация через net use
   - Маппинг буквы диска (опционально)
   - Автоматическое переподключение при сбоях

5. ROBOCOPY
   - Использование robocopy для надёжного копирования
   - Встроенные ретраи
   - Лучшая работа с сетью

6. ОЧЕРЕДЬ ФАЙЛОВ
   - Файлы обрабатываются по порядку (FIFO)
   - При ошибке файл остаётся в очереди
   - Уведомление при переполнении очереди

7. ЛОГИРОВАНИЕ
   - Все события записываются в лог-файл
   - Временные метки для каждого события
   - Логирование ошибок и успешных операций

8. УВЕДОМЛЕНИЯ
   - Email через SMTP
   - HTTP уведомления (Telegram, Slack, webhook)
   - Поддержка прокси для HTTP
   - Логирование всех отправок


ПОТОК ДАННЫХ
------------

Исходная папка → Обнаружение → Сигнатура → Очередь → Стабилизация → 
→ Проверка назначения → Копирование → Удаление из очереди → Лог


ОБРАБОТКА ОШИБОК
----------------

1. Недоступность сетевого диска:
   - Ожидание с интервалом NetworkDriveWaitInterval
   - Попытка переподключения
   - Уведомление администратору

2. Ошибка копирования:
   - Повторные попытки (MaxRetryCount)
   - Задержка между попытками (RetryWaitInterval)
   - Файл остаётся в очереди

3. Файл исчез во время обработки:
   - Удаление из очереди
   - Пропуск файла
   - Следующий проход найдёт файл заново, если он появится

4. Переполнение очереди:
   - Уведомление администратору
   - Продолжение работы
   - Обработка файлов по порядку


КОНФИГУРАЦИЯ
------------

Все параметры в Config.ini:
- Пути к папкам
- Интервалы проверки и ожидания
- Параметры копирования
- Настройки уведомлений
- Параметры SMTP и HTTP

Подробное описание каждого параметра есть в комментариях Config.ini


ЗАПУСК
------

1. Настройка Config.ini
2. Запуск RunFileMonitor.bat
3. Скрипт работает в бесконечном цикле
4. Остановка: Ctrl+C в консоли
