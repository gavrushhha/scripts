================================================================================
ИНСТРУКЦИЯ ПО ЗАПУСКУ СКРИПТА МОНИТОРИНГА ФАЙЛОВ
================================================================================

СОДЕРЖИМОЕ ПРОЕКТА:
-------------------
- FileMonitor.vbs      - основной скрипт мониторинга
- Config.ini            - файл конфигурации (настройки)
- RunFileMonitor.bat    - батник для удобного запуска
- ИНСТРУКЦИЯ_ПО_ЗАПУСКУ.txt - эта инструкция


ШАГ 1: ПОДГОТОВКА КОНФИГУРАЦИИ
--------------------------------
1. Откройте файл Config.ini в любом текстовом редакторе (Блокнот, Notepad++ и т.д.)

2. Настройте обязательные параметры:
   
   а) SourceFolder - укажите папку, которую нужно мониторить
      Пример: SourceFolder=C:\MyFiles\Incoming\
   
   б) DestinationFolder - укажите куда копировать файлы
      Пример: DestinationFolder=\\server\backup\files\
   
   в) CheckInterval - интервал проверки (в миллисекундах)
      Пример: CheckInterval=5000  (проверка каждые 5 секунд)
   
   г) MaxQueueSize - максимальный размер очереди
      Пример: MaxQueueSize=20
   
   д) MaxRetryCount - количество попыток копирования
      Пример: MaxRetryCount=4
   
   е) NetworkDriveWaitInterval - интервал ожидания при недоступности диска
      Пример: NetworkDriveWaitInterval=5000  (5 секунд)
   
   ж) RetryWaitInterval - интервал между повторными попытками
      Пример: RetryWaitInterval=2000  (2 секунды)

3. Настройте параметры аутентификации (если требуется):
   - Username - имя пользователя для сетевого диска
   - Password - пароль для сетевого диска

4. Настройте уведомления (опционально):
   - AdminEmail - email администратора
   - LogFile - путь к файлу лога
   - SMTPServer, SMTPPort и другие параметры SMTP для отправки email

5. Сохраните файл Config.ini


ШАГ 2: ПРОВЕРКА ПРАВ ДОСТУПА
-----------------------------
1. Убедитесь, что у пользователя есть права:
   - На чтение папки SourceFolder
   - На запись в папку DestinationFolder
   - На выполнение скриптов (обычно есть по умолчанию)

2. Если используется сетевой диск:
   - Проверьте доступность сетевого пути
   - Убедитесь, что учетные данные (Username/Password) правильные
   - Проверьте права на запись в сетевую папку


ШАГ 3: ЗАПУСК СКРИПТА
----------------------

СПОСОБ 1: Через батник (РЕКОМЕНДУЕТСЯ)
--------------------------------------
1. Дважды кликните на файл RunFileMonitor.bat
   ИЛИ
2. Откройте командную строку (cmd), перейдите в папку со скриптами:
   cd "C:\путь\к\папке\scripts"
   RunFileMonitor.bat

Преимущества:
- Автоматическая проверка наличия Config.ini
- Правильная кодировка (кириллица)
- Удобный запуск


СПОСОБ 2: Прямой запуск через командную строку
------------------------------------------------
1. Откройте командную строку (Win+R, введите cmd, нажмите Enter)

2. Перейдите в папку со скриптами:
   cd "C:\путь\к\папке\scripts"

3. Запустите скрипт:
   cscript //nologo FileMonitor.vbs

   Или с выводом в консоль:
   cscript FileMonitor.vbs


СПОСОБ 3: Запуск через двойной клик
-------------------------------------
1. Кликните правой кнопкой мыши на FileMonitor.vbs
2. Выберите "Открыть с помощью" -> "Microsoft Windows Based Script Host"
   ИЛИ
3. Просто дважды кликните на FileMonitor.vbs (если .vbs ассоциирован с WScript)

ВНИМАНИЕ: При этом способе могут быть проблемы с кодировкой и выводом сообщений.


ШАГ 4: ПРОВЕРКА РАБОТЫ
-----------------------
1. После запуска вы должны увидеть сообщения в консоли:
   "Мониторинг папки запущен: [путь]"
   "Целевая папка: [путь]"

2. Скрипт начнет проверять папку SourceFolder на наличие новых файлов

3. При обнаружении нового файла:
   - Файл будет добавлен в очередь
   - Будет вычислен хэш файла
   - Файл будет скопирован на DestinationFolder
   - Хэши будут сравнены
   - При успехе файл будет удален из очереди

4. Проверьте лог-файл (если указан в Config.ini):
   - Откройте файл, указанный в параметре LogFile
   - Там должны быть записи о работе скрипта

5. Проверьте email (если настроен):
   - При проблемах должны приходить уведомления на AdminEmail


ОСТАНОВКА СКРИПТА
-----------------
1. Если скрипт запущен в консоли:
   - Нажмите Ctrl+C
   - Подтвердите остановку

2. Если скрипт запущен через двойной клик:
   - Закройте окно скрипта
   - Или завершите процесс через Диспетчер задач (wscript.exe или cscript.exe)


РЕШЕНИЕ ПРОБЛЕМ
---------------

Проблема: "Файл конфигурации не найден"
Решение: Убедитесь, что Config.ini находится в той же папке, что и FileMonitor.vbs

Проблема: "ОШИБКА: В конфигурации отсутствуют обязательные параметры"
Решение: Проверьте Config.ini, все обязательные параметры должны быть заполнены

Проблема: "Ошибка доступа к папке"
Решение: Проверьте права доступа и существование папки SourceFolder

Проблема: "Сетевой диск недоступен"
Решение: 
- Проверьте доступность сетевого пути
- Проверьте учетные данные (Username/Password)
- Проверьте сетевое подключение

Проблема: "Ошибка отправки email"
Решение:
- Проверьте настройки SMTP (SMTPServer, SMTPPort)
- Проверьте учетные данные SMTP (SMTPUser, SMTPPassword)
- Проверьте доступность SMTP сервера

Проблема: Скрипт не запускается
Решение:
- Убедитесь, что установлен Windows Script Host
- Попробуйте запустить через командную строку с правами администратора
- Проверьте, не блокирует ли антивирус скрипт


АВТОМАТИЧЕСКИЙ ЗАПУСК ПРИ ЗАГРУЗКЕ WINDOWS
-------------------------------------------
1. Нажмите Win+R, введите: shell:startup
2. Создайте ярлык на RunFileMonitor.bat в этой папке
3. При загрузке Windows скрипт будет запускаться автоматически

ИЛИ

1. Откройте Планировщик заданий (taskschd.msc)
2. Создайте новое задание
3. Триггер: При входе в систему
4. Действие: Запустить программу
5. Программа: C:\Windows\System32\cscript.exe
6. Аргументы: //nologo "C:\путь\к\FileMonitor.vbs"
7. Начало в: папка со скриптами


ДОПОЛНИТЕЛЬНАЯ ИНФОРМАЦИЯ
--------------------------
- Все настройки находятся в файле Config.ini
- Подробное описание каждого параметра есть в комментариях Config.ini
- Логи работы сохраняются в файл, указанный в параметре LogFile
- Уведомления отправляются на email, указанный в AdminEmail
- Скрипт работает в бесконечном цикле до ручной остановки
